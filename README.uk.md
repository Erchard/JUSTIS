# JUSTIS

JUSTIS - відкритий проєкт зі створення Універсального кодексу справедливості через дисципліновані дискусії, заперечення, граничні кейси й переглядні рішення.

Це не маніфест остаточної істини. Це робочий корпус для перевірки моральних принципів.

## Читати Спочатку

1. `codex/UNIVERSAL_JUSTICE_CODE.md` - латинський канонічний Кодекс.
2. `codex/translations/UNIVERSAL_JUSTICE_CODE.uk.md` - український переклад Кодексу.
3. `codex/translations/` - інші офіційні переклади.
4. `OPEN_QUESTIONS.md` - що ще треба розглянути.
5. `DECISIONS.md` - що вже вирішено.
6. `INDEX.md` - коротка карта проєкту.

## Робоче Ядро

- `codex/` - лише канонічний Кодекс і його офіційні переклади.
- `support/codex/` - статуси, обґрунтування, терміни, політика перекладів і розгорнуті записи законів.
- `protocols/` - повні протоколи дискусій Небесного парламенту.
- `arguments/` - дерева аргументів і контраргументів.
- `research/` - робочі дослідження аксіом.
- `cases/` і `CASES.md` - тестові морально-правові кейси.
- `sources/` і `specifications/AUTHORITATIVE_SOURCES.md` - джерела.
- `specifications/` - чинні методологічні стандарти і склад парламенту.
- `templates/` - шаблони для нових документів.
- `tools/` - технічні перевірки корпусу.
- `archive/` - матеріали, які збережені для історії, але не є активним робочим входом.

## Метод

1. Формулюємо питання.
2. Реконструюємо сильні позиції різних авторитетів.
3. Збираємо заперечення і відповіді.
4. Перевіряємо граничні випадки й ризики зловживання.
5. Якщо консенсус достатній, фіксуємо закон Кодексу.
6. Якщо ні, фіксуємо апорію або повертаємо питання в беклог.

## Поточний Стан

У Кодексі зафіксовано `Lex I-V`.

Q-006, питання про те, чи може справедливість існувати без правди, вже має статус `protocol_done`, але ще не внесене до Кодексу.

Наступний змістовний крок: перевірити Q-006 на готовність до `codex_candidate` або явно позначити його як `revisendum` / `aporia`.

## Правило Змін

Після кожної зміни файлів робиться окремий Git commit. Старі матеріали не видаляються без потреби: їх краще переносити в `archive/` з поясненням.
