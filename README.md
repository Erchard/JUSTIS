# Універсальний кодекс справедливості

**JUSTIS — відкритий, придатний для читання агентами проект зі створення Універсального кодексу справедливості через структуровані дискусії, заперечення, граничні випадки й ратифіковані принципи. Це не маніфест остаточної істини, а публічна процедура перевірки тверджень про справедливість.**

Цей проект присвячений створенню Універсального кодексу справедливості: системи принципів, які можуть слугувати мірилом для законів, інституцій, суспільних практик і особистих рішень.

Проект не намагається одразу проголосити готову істину. Він будується як дисципліноване дослідження: через конфліктні теми, аргументи, контраргументи, перевірку аксіом, граничні випадки й рішення Небесного парламенту.

## Робоча назва дискусійного клубу

**Небесний парламент** - уявний дискусійний клуб авторитетних мислителів, традицій і моральних перспектив. Його авторитет походить не з непомильності учасників, а з сили аргументів, чесності контраргументів і готовності переглядати висновки.

## Структура проекту

- `OPEN_QUESTIONS.md` - реєстр усіх проблемних питань, які потрібно розглянути.
- `INDEX.md` - короткий маршрут читання для людей, критиків, дослідників і агентів.
- `DECISIONS.md` - реєстр прийнятих рішень Небесного парламенту.
- `CONTRIBUTING.md` - як людям долучатися до проекту.
- `AGENTS.md` - як нейромережевим агентам працювати з проектом.
- `GOOD_FIRST_CONTRIBUTIONS.md` - прості перші внески для людей і агентів.
- `PROJECT_BRIEF.en.md` - короткий англомовний вступ.
- `llms.txt` - стислий вхід для нейромережевих агентів.
- `agent_tasks.yml` - структуровані задачі для агентів.
- `CITATION.cff` - метадані для цитування проекту.
- `LICENSE.md` - ліцензійні правила проекту.
- `RELEASE_NOTES_v0.1.0.md` - підготовлений опис першого публічного релізу.
- `PROJECT_AUDIT.md` - аудит організації проекту.
- `PROJECT_PRINCIPLES_AUDIT.md` - аудит відповідності принципам проекту.
- `specifications` - методологія, технічне завдання, карти джерел і конфліктних тем.
- `protocols` - протоколи дискусій, рішень і статусів консенсусу.
- `research` - робочі дослідження аксіом, тем і граничних випадків.
- `sources` - нотатки по джерелах, авторам і традиціях.
- `cases` - тестові морально-правові кейси для перевірки Кодексу.
- `codex` - майбутні тексти самого кодексу, декларації, законів і принципів.
- `templates` - шаблони для протоколів, досліджень, джерельних нотаток і законів.
- `archive` - застарілі або замінені матеріали, які не треба втрачати.
- `tools` - допоміжні скрипти для перевірки цілісності проекту.

## Fork-friendly project

Цей проект можна форкати.

Форки можуть пропонувати альтернативну структуру, інші формулювання законів, нові кейси, заперечення, джерела, переклади й агентські протоколи.

Найцінніший внесок - не згода з автором, а сильне заперечення, яке робить Кодекс точнішим, чеснішим і менш вразливим до зловживання.

Почати можна з `GOOD_FIRST_CONTRIBUTIONS.md`.

## Поточний метод

1. Перевіряємо базові аксіоми.
2. Обираємо конфліктну тему.
3. Реконструюємо найсильніші позиції сторін.
4. Збираємо контраргументи.
5. Перевіряємо граничні випадки.
6. Застосовуємо тести зловживання, реальності, публічної прийнятності, незворотної шкоди, чесноти, спільного блага і прихованих мотивів.
7. Фіксуємо результат як закон, презумпцію, виняток, правило зважування або апорію.
8. Залишаємо умови перегляду.

## Правило збереження змін

Після кожної зміни файлів проекту робиться окремий Git commit. Це дає змогу повернутися до будь-якого етапу роботи.

## Як долучатися

Проект відкритий до питань, джерел, аргументів, контраргументів і граничних випадків.

Почати можна з:

- `CONTRIBUTING.md` - для людей;
- `AGENTS.md` - для нейромережевих агентів;
- `GOOD_FIRST_CONTRIBUTIONS.md` - для перших простих внесків;
- `OPEN_QUESTIONS.md` - щоб побачити, які питання чекають розгляду;
- `codex/UNIVERSAL_JUSTICE_CODE.md` - щоб побачити короткий поточний Кодекс.
