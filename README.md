# JUSTIS

JUSTIS is an open project for building a Universal Code of Justice through structured deliberation, objections, edge cases, and revisable decisions.

It is not a manifesto of final truth. It is a public working corpus for testing moral claims.

English readers should begin with:

1. `PROJECT_BRIEF.en.md` - the shortest English brief.
2. `README.en.md` - the main English overview.

7. `codex/translations/UNIVERSAL_JUSTICE_CODE.en.md` - the English translation of the Code.

Ukrainian remains the primary working language of many protocols and research notes, but the public entrance to the project is now English-readable.

## What This Is

JUSTIS tries to discover principles of justice that are not reducible to the law of one country, the interest of one group, or the preference of one person.

The project works by:

1. posing a morally important question;
2. reconstructing strong positions from authoritative traditions;
3. collecting objections and counter-objections;
4. testing the principle against edge cases;
5. recording unresolved tensions instead of hiding them;
6. adding a principle to the Code only when the procedure supports it.

## What This Is Not

JUSTIS is not:

- a political party;
- a religion;
- a finished ethical doctrine;
- a vote-counting parliament;
- an attempt to replace law with private opinion.

The project values consensus, traceability, and strong objections more than speed.

## Active Core

- `codex/` - only the canonical Code and its official translations.
- `support/codex/` - status indexes, justifications, terminology, translation policy, and detailed law records.
- `protocols/` - deliberation records and decisions.
- `arguments/` - structured argument and counterargument trees.
- `research/` - working research notes.
- `cases/` and `CASES.md` - moral and legal test cases.
- `sources/` and `specifications/AUTHORITATIVE_SOURCES.md` - source material.
- `specifications/` - methodology, procedural standards, membership, and JML.
- `templates/` - templates for new documents.
- `tools/` - corpus validation tools.
- `archive/` - preserved historical or non-active materials.

## Current State

The Code currently contains `Lex I-V`.

The next recommended substantive question is Q-006: whether justice can exist without truth.

## Change Rule

Every file change should be committed to Git. Old material should usually be moved to `archive/` instead of being deleted when it may still have historical value.
