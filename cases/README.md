# Тестові кейси

Папка `cases/` зберігає окремі картки складних морально-правових ситуацій.

Кожен кейс має відповідати шаблону `templates/case-test.md` і бути зареєстрований у `CASES.md`.

## Типи кейсів

- `judicial_case` - судове рішення або квазісудове рішення.
- `historical_case` - історична ситуація з достатнім описом фактів.
- `hypothetical_case` - гіпотетична моральна дилема.
- `literary_case` - сюжет із літератури або філософського прикладу.
- `policy_case` - публічна політика або законодавча дилема.

## Мінімум для картки

Картка має містити:

- короткий опис фактів;
- моральний конфлікт;
- зацікавлені або вразливі сторони;
- які принципи Кодексу тестуються;
- рішення суду або фактичний результат, якщо є;
- попередню оцінку Кодексу;
- умови перегляду.

## Обережність

Судове рішення не є автоматично справедливим. Воно є добре документованим матеріалом для тесту.

Кодекс також не має автоматично заперечувати суд. Якщо Кодекс і суд розходяться, потрібно окремо з'ясувати: помиляється суд, помиляється Кодекс, чи конфлікт показує глибше питання.
