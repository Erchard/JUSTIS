# Реєстр відкритих проблемних питань

## Призначення

Цей документ фіксує всі проблемні питання, які потрібно розглянути в межах проекту "Універсальний кодекс справедливості".

Реєстр потрібен, щоб:

- не втрачати теми, які відкриваються під час дискусій;
- бачити пріоритети;
- відрізняти аксіоми від прикладних тем;
- знати статус кожного питання;
- планувати порядок денний сесій Небесного парламенту.

## Статуси

- `not_started` - питання ще не досліджувалося.
- `opened` - питання сформульоване, але не опрацьоване.
- `in_research` - створено дослідницький файл або йде робота.
- `in_session` - питання винесено на сесію Небесного парламенту.
- `protocol_done` - є протокол рішення.
- `argument_tree_done` - є дерево аргументів.
- `codex_candidate` - готове до внесення в Кодекс.
- `codex_done` - внесено в Кодекс.
- `aporia` - зафіксована нерозв'язана напруга.

## Пріоритети

- `P0` - базове питання, без якого інші теми залежать від неперевіреної основи.
- `P1` - ключове питання для Кодексу.
- `P2` - важливе питання для окремих сфер застосування.
- `P3` - додаткове або спеціалізоване питання.

## Правило додавання нових питань

Нове питання додається до реєстру, якщо воно:

- відкриває глибшу розбіжність;
- блокує поточне рішення;
- потрібне для майбутнього закону Кодексу;
- виявляє ризик зловживання;
- показує недопредставлений голос;
- потребує окремого протоколу або дерева аргументів.
