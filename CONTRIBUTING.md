# Як долучитися

Проект відкритий до внесків, але працює не як голосування думок, а як дисциплінована дискусія.

## Центральна ідея (для всіх нових агентів і контриб’юторів)

Нам **не** потрібен текст “від одного агента”. Нам потрібна **жива гостра дискусія** Небесного парламенту: **62 авторитетні історичні мислителі та традиції** з **протилежними** поглядами перевіряють кожне правило через аргументи, сильні заперечення, граничні кейси та практичні наслідки.

Тому внесок має підсилювати **колегіальність**:

- додайте тезу **від імені конкретного мембра/традиції** або з явним `fides_fontis`;
- додайте **сильне заперечення** (не “солом’яне”), бажано від іншої традиції;
- додайте **відповідь** або позначте заперечення як відкрите з умовами перегляду.

## Що можна запропонувати

- нове питання для `OPEN_QUESTIONS.md`;
- джерело або автора;
- аргумент;
- контраргумент;
- граничний випадок;
- приклад несправедливості;
- свідчення вразливої сторони;
- виправлення помилки;
- уточнення формули закону;
- дерево аргументів;
- чернетку протоколу.

Для простого старту дивіться `INDEX.md`, `OPEN_QUESTIONS.md` і `.github/ISSUE_TEMPLATE/`.

Якщо ви працюєте через GitHub, можна скористатися issue forms:

- поставити моральне питання;
- подати заперечення;
- подати кейс;
- запропонувати джерело;
- запропонувати зміну структури;
- подати внесок нейромережевого агента.

## Як має виглядати хороший внесок

Добрий внесок:

- чітко вказує, до якого питання або закону належить;
- відрізняє факт, цитату, тлумачення і припущення;
- формулює найсильнішу версію позиції;
- не карикатурить опонента;
- показує, яке рішення або статус має змінитися;
- додає умови перегляду, якщо це потрібно.

## Мінімальний формат аргументу

```yaml
typus: argumentum
quaestio: "Q-000"
thesis: "Коротка теза"
ratio: "Чому це важливо"
fontes:
  - "Джерело або автор"
fides_fontis: direct_text / strong_interpretation / reconstruction / analogy / speculative
objectiones:
  - "Сильне заперечення"
```

## Мінімальний формат контраргументу

```yaml
typus: obiectio
respondet_ad: "argumentum або actus"
thesis: "Коротке заперечення"
ratio: "Що саме воно ламає або уточнює"
status_propositus: "soluta / non_soluta / partialiter_soluta"
```

## Важливе правило

Жоден внесок не стає законом Кодексу автоматично.

Він має пройти:

1. фіксацію питання;
2. аргументи й контраргументи;
3. повний протокол або оновлення дерева аргументів;
4. рішення Небесного парламенту;
5. запис у `DECISIONS.md` і, якщо потрібно, в `codex`.

## Межі канону (щоб не було плутанини)

- **`codex/` містить тільки канонічний текст Кодексу і офіційні переклади.**
- **Ні агент, ні автор PR не “канонізує” закон сам по собі.**

Практичний STOP-лист для внесків:

- не додавайте новий Lex у `codex/` без відповідного протоколу в `protocols/` і запису в `DECISIONS.md`;
- не позначайте питання як `codex_done`, якщо воно не внесене в `codex/UNIVERSAL_JUSTICE_CODE.md`;
- якщо ваш внесок є лише чернеткою або реконструкцією — позначайте це через `fides_fontis` і не підвищуйте статус “консенсусу” без видимих заперечень і відповідей.
