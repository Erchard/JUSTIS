# Методика тестування Кодексу на кейсах

## Призначення

Кодекс справедливості має перевірятися не лише на абстрактних аргументах, а й на складних реальних ситуаціях.

Кейс-тестування потрібне, щоб виявляти:

- надто широкі формули;
- приховані винятки;
- конфлікти між законами Кодексу;
- ризики зловживання;
- культурні або інституційні упередження;
- ситуації, де Кодекс дає кращу відповідь, ніж позитивне право;
- ситуації, де позитивне право виявляє слабкість Кодексу.

## Джерела кейсів

Перший пріоритет мають джерела, де факти й рішення вже структуровані:

- міжнародні суди;
- регіональні суди з прав людини;
- договірні органи ООН;
- національні реєстри судових рішень;
- спеціалізовані бази з медицини, прав людини, міжнародного кримінального права;
- класичні моральні дилеми з філософії, якщо вони потрібні для чистого тесту принципу.

## Критерії добору

Кейс варто додавати, якщо він:

1. Має чіткий моральний конфлікт.
2. Стосується відкритого питання або закону Кодексу.
3. Містить напругу між щонайменше двома цінностями.
4. Має достатньо фактів для чесного аналізу.
5. Може спростувати або уточнити принцип.

Кейс не варто додавати, якщо він:

- є лише ілюстрацією без реальної напруги;
- має надто мало фактів;
- використовується тільки для підтвердження вже бажаного висновку;
- потребує спеціальної експертизи, якої ми ще не маємо.

## Процедура аналізу

1. Зафіксувати факти максимально нейтрально.
2. Визначити сторони, які можуть виграти або постраждати.
3. Сформулювати моральний конфлікт.
4. Вказати, які закони або відкриті питання Кодексу активуються.
5. Порівняти рішення суду або фактичний результат із можливою відповіддю Кодексу.
6. Зафіксувати напругу:
   - Кодекс підтверджує рішення;
   - Кодекс критикує рішення;
   - Кодекс не має достатнього принципу;
   - Кодекс дає суперечливі сигнали.
7. Визначити наслідок для проекту:
   - без змін;
   - уточнити закон;
   - додати виняток;
   - додати ризик зловживання;
   - додати нове відкрите питання;
   - повернути тему в беклог.

## Мінімальна JML-структура кейсу

```yaml
id: casus.C-000
typus: casus
status: candidate
titulus: "Назва кейсу"
fons: "База або джерело"
quaestiones:
  - "Q-000"
leges_probatae:
  - "lex.000"
conflictus:
  - "цінність 1"
  - "цінність 2"
exitus:
  iudicium: "коротко про рішення суду, якщо є"
  codex: "попередня відповідь Кодексу"
status_examinationis: "not_tested"
```

## Рівні результату

- `codex_confirmed` - кейс підтримує поточну формулу Кодексу.
- `codex_refined` - кейс змусив уточнити формулу або коментар.
- `codex_conflict_detected` - кейс виявив конфлікт між принципами.
- `codex_gap_detected` - Кодексу бракує принципу.
- `case_inconclusive` - фактів недостатньо або кейс не дає ясного результату.

## Зв'язок із сесіями

Кейс може бути:

- підготовчим матеріалом до сесії;
- граничним тестом після формулювання принципу;
- причиною перегляду вже прийнятого закону;
- підставою для відкриття нового питання в `OPEN_QUESTIONS.md`.

Для Q-002 про гідність перший рекомендований набір:

1. Кейс про нелюдське або принизливе поводження.
2. Кейс про використання людини як засобу заради більшості.
3. Кейс про конфлікт гідності зі свободою слова або безпекою.
