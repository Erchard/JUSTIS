# Структура проекту

## 0. Кореневі документи

Корінь проекту містить загальні навігаційні й управлінські документи:

- `README.md` - короткий вступ;
- `CONTRIBUTING.md` - правила участі для людей;
- `AGENTS.md` - правила участі для нейромережевих агентів;
- `INDEX.md` - маршрут читання для різних типів учасників;
- `GOOD_FIRST_CONTRIBUTIONS.md` - прості перші внески для людей і агентів;
- `PROJECT_BRIEF.en.md` - короткий англомовний вступ до проекту;
- `llms.txt` - стислий маршрут читання для нейромережевих агентів;
- `agent_tasks.yml` - структуровані задачі для нейромережевих агентів;
- `CITATION.cff` - метадані для цитування проекту;
- `LICENSE.md` - ліцензійні правила проекту;
- `RELEASE_NOTES_v0.1.0.md` - підготовлений опис першого публічного релізу;
- `PROJECT_STRUCTURE.md` - карта структури;
- `PROJECT_AUDIT.md` - аудит організації проекту;
- `PROJECT_PRINCIPLES_AUDIT.md` - аудит відповідності принципам проекту;
- `OPEN_QUESTIONS.md` - реєстр відкритих проблемних питань;
- `DECISIONS.md` - реєстр прийнятих рішень Небесного парламенту;
- `CASES.md` - реєстр тестових морально-правових кейсів.

## 1. `specifications`

Папка для стабільних проектних документів:

- технічне завдання;
- методологія;
- корпус авторитетних джерел;
- карта конфліктних тем;
- критика методології;
- рішення щодо назв і базових понять.
- специфікація Justice Markup Language.
- стандарт процедурної чесності.
- стратегія користі, публічності й участі.
- методика тестування Кодексу на складних кейсах.

Документи в цій папці описують рамку проекту.

## 2. `protocols`

Папка для протоколів Небесного парламенту.

Вона розділена на моральні й організаційні протоколи, щоб не змішувати зміст справедливості з правилами організації пошуку справедливості.

Сюди належать:

- моральні протоколи про аксіоми, конфліктні теми й кейси;
- організаційні протоколи про процедуру, склад парламенту, архітектуру Кодексу й участь;
- прийняті рішення;
- статуси консенсусу;
- процедурна надійність рішень;
- умови перегляду;
- короткі підсумки голосів авторитетів.

Поточна структура:

```text
protocols/
  moral/
    axioms/
    topics/
    cases/
  organizational/
    procedure/
    membership/
    codex/
    outreach/
  README.md
  RATIFICATION_REGISTER.md
```

`protocols/RATIFICATION_REGISTER.md` лишається в корені папки як спільний службовий реєстр.

Рекомендована назва файлу:

```text
YYYY-MM-DD_short-topic.md
```

## 3. `research`

Папка для робочих досліджень.

### `research/axioms`

Дослідження базових аксіом:

- страждання;
- гідність;
- сила і право;
- універсалізація;
- влада і примус;
- справедливість вище позитивного закону.

### `research/topics`

Дослідження прикладних і конфліктних тем:

- війна;
- брехня;
- власність;
- покарання;
- свобода слова;
- традиція і права людини;
- майбутні покоління.

## 4. `sources`

Папка для нотаток по джерелах.

Кожне джерело бажано оформлювати окремим файлом:

```text
author-short-title.md
```

Приклади:

```text
kant-groundwork.md
rawls-theory-of-justice.md
kung-global-ethic.md
```

## 5. `codex`

Папка для майбутнього тексту кодексу.

Очікувані файли:

- `UNIVERSAL_JUSTICE_CODE.md` - лаконічний перелік законів Кодексу без службових статусів і обґрунтувань;
- `UNIVERSAL_JUSTICE_DECLARATION.md` - коротка декларація;
- `STATUS_INDEX.md` - службовий індекс статусів законів;
- `JUSTIFICATION_INDEX.md` - службовий індекс обґрунтувань і простежуваності;
- `TERMINOLOGY.md` - пояснення різниці між аксіомою, принципом і законом Кодексу;
- `LAWS.md` - розгорнуті записи законів кодексу;
- `PRESUMPTIONS.md` - презумпції;
- `WEIGHING_RULES.md` - правила зважування;
- `GLOSSARY.md` - словник понять.

## 6. `templates`

Папка шаблонів.

Шаблони допомагають не винаходити структуру щоразу заново.

## 7. `arguments`

Папка для структурованих дерев аргументів.

Вона потрібна, щоб фінальний `codex` міг залишатися лаконічним, а всі докази, контраргументи, граничні випадки й умови перегляду зберігалися окремо.

Підпапки:

- `arguments/laws` - дерева аргументів для законів кодексу;
- `arguments/axioms` - дерева аргументів для базових аксіом;
- `arguments/topics` - дерева аргументів для конфліктних тем.

## 8. `cases`

Папка для окремих карток тестових морально-правових кейсів.

Кейси використовуються як полігон перевірки Кодексу:

- чи працює закон на реальних складних ситуаціях;
- чи виникають винятки;
- чи є ризик зловживання принципом;
- чи треба уточнювати Кодекс або дерево аргументів.

Кожен кейс має бути зареєстрований у `CASES.md` і, якщо можливо, оформлений за шаблоном `templates/case-test.md`.

## 9. `archive`

Папка для матеріалів, які втратили актуальність, але не повинні бути видалені.

Архів не є смітником. Якщо файл переноситься сюди, бажано коротко пояснити причину.

## 10. `tools`

Папка для допоміжних технічних інструментів проекту.

Поточний інструмент:

- `tools/validate_project.py` - легка перевірка реєстрів, статусів, посилань і простежуваності законів Кодексу.

Ці інструменти не приймають моральних рішень. Вони лише ловлять технічні розсинхрони.

## 11. Git-дисципліна

Після кожної зміни файлів потрібно робити commit.

Бажаний стиль повідомлень:

```text
Add ...
Update ...
Refine ...
Move ...
Archive ...
```

Приклади:

```text
Add initial axiom research template
Update methodology with burden of proof
Refine Heavenly Parliament naming protocol
```

## 12. `.github`

Папка для GitHub-інфраструктури зовнішньої участі.

Містить:

- issue forms для питань, заперечень, кейсів, джерел, структурних пропозицій і агентських внесків;
- pull request template з перевіркою процедурної чесності.

Ця папка не замінює регламент Небесного парламенту. Вона лише робить вхід зовнішніх учасників структурованішим.
