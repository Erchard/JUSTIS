# Протоколи дискусій і рішень

## Призначення папки

У цій папці зберігаються протоколи дискусій Небесного парламенту, прийняті рішення, статуси консенсусу, відкриті заперечення і підстави для майбутнього перегляду.

Ця папка відрізняється від `specifications`:

- `specifications` містить методологію, джерела, технічне завдання і загальні карти проекту;
- `protocols` містить конкретні дискусії, рішення і протоколи голосу авторитетів.

## Що сюди зберігати

- протоколи обговорення назв, методів, аксіом і принципів;
- рішення Небесного парламенту;
- статуси результатів: повний консенсус, обмежений консенсус, презумпція, апорія, правило зважування;
- ключові аргументи сторін;
- контраргументи;
- умови перегляду рішення;
- посилання на пов'язані специфікації.

## Рекомендований формат назви файлів

```text
YYYY-MM-DD_short-topic.md
```

Приклади:

```text
2026-04-26_heavenly-parliament-name.md
2026-04-26_methodology-second-reading.md
2026-04-26_axioms-initial-review.md
```

## Формат протоколу

Нові протоколи мають використовувати повний дискусійний формат із JML-блоком, виступами, запереченнями, відповідями і матрицею аргументів.

Мінімальний стандарт описаний у:

- `templates/deliberation-protocol.md`
- `specifications/PROCEDURAL_INTEGRITY_STANDARD.md`
- `protocols/RATIFICATION_REGISTER.md`

Старі протоколи, які є лише резюме, не видаляються. Вони зберігаються як історія проекту, але в `DECISIONS.md` мають отримати процедурний статус `summarium_provisorium`, `deliberatio_partialis` або `ratificatio_requiritur`.

```markdown
# Назва протоколу

## Дата

## Тема

## Питання

## Позиції

## Аргументи

## Контраргументи

## Рішення

## Статус

## Умови перегляду

## Пов'язані документи
```

## Заборона удаваного консенсусу

Якщо протокол не містить видимих аргументів, заперечень і відповідей, він не може сам по собі підтверджувати `consensus_plenus`.

У такому випадку рішення може бути:

- робочим;
- попереднім;
- практичним;
- таким, що потребує ратифікації;
- або апорією.

## Статус папки

Папка створена як робоче місце для протоколів Небесного парламенту.

Починаючи з аудиту процедурної чесності, нові рішення мають фіксуватися у повному дискусійному форматі або явно позначатися як технічні/редакційні.
