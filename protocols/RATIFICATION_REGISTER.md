# Реєстр ратифікації попередніх рішень

## Призначення

Цей документ показує, які рішення вже мають повну процедурну основу, а які були створені як резюме або часткова дискусія і потребують повторного розгляду.

Реєстр не скасовує попередні рішення. Він допомагає чесно керувати їхнім статусом.

## Статуси

- `ratified` - має повну дискусійну структуру.
- `priority_ratification` - потрібно ратифікувати першочергово.
- `ratification_required` - потребує ратифікації, але не першим кроком.
- `editorial_or_historical` - може залишатися попереднім, якщо не впливає на зміст Кодексу.

## Перша черга ратифікації

Рекомендована перша черга:

1. D-004 - правило консенсусу.
2. D-013 - формалізація через DSL.
3. D-006 - регламент сесій.
4. D-012 - метаблок сесії.
5. D-003 - архітектура Кодексу.

D-010 залишається важливим рішенням для другої черги ратифікації, але не блокує старт роботи з першою аксіомою після завершення мінімального процедурного каркасу.

## Правило роботи

Поки рішення має статус `ratification_required` або `priority_ratification`, його можна використовувати як робочу норму, але не як остаточну підставу для внесення закону до Кодексу.

Для кожної ратифікації створюється новий повний протокол у форматі `templates/deliberation-protocol.md`.
